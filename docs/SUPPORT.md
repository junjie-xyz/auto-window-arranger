# Support / 支持

Auto Window Arranger is maintained by independent developer Ding Junjie (丁俊杰). / Auto Window Arranger 由独立开发者丁俊杰维护。

## English

### Current channels

- Public, non-sensitive problems: [GitHub Issues](https://github.com/junjie-xyz/auto-window-arranger/issues)
- Private support and privacy requests: `ioriens@outlook.com`
- General product contact: [@junjie_xyz](https://x.com/junjie_xyz)
- Security reports: follow [SECURITY.md](../SECURITY.md). Do not use a public issue for vulnerabilities or sensitive logs.

### What to include

For faster troubleshooting, include:

- Auto Window Arranger version and build;
- macOS version and Mac architecture;
- the action you performed, expected result, and actual result;
- the smallest repeatable steps;
- a sanitized report from **Copy Diagnostics**, if relevant.

Do not include passwords, tokens, certificates, private keys, window contents, user input, private file contents, or unredacted logs. The diagnostic report is copied to your clipboard and is sent only if you choose to paste or attach it.

### Scope

The current release target is macOS 14 or later. Apple Silicon is supported. A Universal Binary may include Intel code before Intel hardware validation is complete; public release notes will identify Intel as unverified until that validation exists.

Before the first stable release, no response-time or long-term maintenance commitment is offered. Release-specific supported versions and known limitations will be published with each stable release.

## 简体中文

### 当前渠道

- 公开的非敏感问题：[GitHub Issues](https://github.com/junjie-xyz/auto-window-arranger/issues)
- 私密支持和隐私请求：`ioriens@outlook.com`
- 一般产品联系：[@junjie_xyz](https://x.com/junjie_xyz)
- 安全报告：按照 [SECURITY.md](../SECURITY.md) 处理。不要通过公开 Issue 提交漏洞或敏感日志。

### 提交内容

为便于排查，请提供：

- Auto Window Arranger 版本和 build；
- macOS 版本和 Mac 架构；
- 执行的操作、预期结果和实际结果；
- 最小可重复步骤；
- 如有需要，提供“复制诊断信息”生成的脱敏报告。

不要提交密码、token、证书、私钥、窗口内容、用户输入、私密文件内容或未脱敏日志。诊断报告只会复制到剪贴板，只有你主动粘贴或附加时才会发送。

### 支持范围

当前发布目标为 macOS 14 及以上版本。Apple Silicon 是受支持平台。Universal Binary 可能在 Intel 真机验证完成前包含 Intel 代码；完成验证前，公开发布说明会把 Intel 标记为“尚未验证”。

首个 stable 发布前不承诺响应时间或长期维护周期。每个 stable 版本会单独公布受支持版本和已知限制。
